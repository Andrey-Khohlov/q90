# consequences
Единое место определения — логика валидации в одном классе
Переиспользование — можно использовать в разных моделях
Гибкость — работаем и со строками, и с enum'ами
Чистый код — модели не загромождены валидацией
Самодокументируемость — тип поля сразу показывает допустимые значения
Практически бесплатный способ гарантировать чистоту данных: CHECK-ограничение проверяется исключительно в момент вставки или изменения строки. Оно никак не влияет на скорость чтения (SELECT).
