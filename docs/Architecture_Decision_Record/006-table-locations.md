# Бизнес-правила иерархии geography.locations

### Пример потока
Пользователь начинает вводить название страны (например, "Кения").
Система предлагает варианты, пользователь выбирает.
Появляется список регионов Кении (если есть). Пользователь выбирает, например, "Киамбу".
Далее — выбор locality/city (например, "Тхика").
В итоге: часть света "Африка" автоматически проставляется.

### Допустимые связи parent–child
| Тип потомка    | Допустимые типы родителя                | Примечание                          |
|----------------|------------------------------------------|-------------------------------------|
| `world_part`   | `NULL` (корень)                          | Части света – верхний уровень      |
| `country`      | `world_part`                             | Страна принадлежит части света     |
| `region`       | `country`, `region`                       | Регион может входить в страну или другой регион (адм. деление) |
| `locality`     | `region`, `locality`                      | Населенный пункт или местность может быть вложенной |
| `city`         | `locality`, `region`                      | Город может относиться к местности или напрямую к региону |

**Важно:** Никакая запись не может ссылаться на тип, не указанный в таблице. Запрещены циклы (например, регион, ссылающийся на свой дочерний узел).

### Ограничения на атрибуты в зависимости от типа
- **`country_code`**  
  - Для `country`: обязательно (`NOT NULL`) и уникален среди всех стран (частичный уникальный индекс).  
  - Для остальных типов: должен быть `NULL`.
- **`is_coffee_growing`**  
  - Имеет смысл только для `country`; для остальных типов игнорируется (может быть `false` или `NULL` по соглашению, но предпочтительно `false`).
- **`parent_id`**  
  - Может быть `NULL` только для `world_part` (корневые элементы). Все остальные типы обязаны иметь родителя.

### Целостность ссылок
- Не допускается ссылка на удалённую запись (`deleted_at IS NOT NULL`). Приложение должно учитывать мягкое удаление при выборе родителя.
- Каскадное удаление не настроено – удаление записи, имеющей потомков, приведёт к ошибке внешнего ключа. Для удаления узла необходимо сначала переназначить или удалить всех потомков.

### Рекомендации по работе с данными
1. Все операции вставки/обновления должны проверять допустимость типа родителя согласно таблице выше. Желательно реализовать эту проверку на уровне БД (триггер) или в приложении.
2. Уникальность `country_code` обеспечивается частичным индексом; дубликаты недопустимы.
3. При изменении `parent_id` убедитесь, что не создаётся цикл (например, рекурсивная проверка).
4. Для запросов иерархии используйте рекурсивные CTE с индексом по `parent_id`.

Эти правила гарантируют логическую целостность географической иерархии и предотвращают случайное нарушение структуры данных.

```sql
WITH RECURSIVE location_path AS (
    -- Стартуем с заданного города
    SELECT location_id, name, location_type, parent_id
    FROM geography.locations
    WHERE location_id = :city_id AND location_type = 'city'

    UNION ALL

    -- Рекурсивно поднимаемся по parent_id
    SELECT l.location_id, l.name, l.location_type, l.parent_id
    FROM geography.locations l
    INNER JOIN location_path lp ON l.location_id = lp.parent_id
)
SELECT location_id, name, location_type
FROM location_path
WHERE location_type IN ('world_part', 'country', 'region', 'locality');
```
